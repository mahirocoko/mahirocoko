# mahiro-style examples

These examples exist to make the doctrine concrete.

Use them when the principle alone is too abstract and you need to show what Mahiro-style code should or should not look like.

## Example Map

- `interface-vs-type.md` - when to use `interface` vs `type`
- `route-boundaries.md` - thin route orchestration vs giant route files
- `constants-i18n.md` - extracting config without breaking Lingui flow
- `naming.md` - domain naming vs generic names
- `shared-ui-boundaries.md` - reusable UI vs page-specific logic
- `services.md` - service-layer intent vs transport leakage
- `stores.md` - store boundaries vs dumping-ground state
- `section-order.md` - predictable component and hook ordering
- `export-style.md` - export choices that match local repo conventions

## Reading Mode

- `Do` = closer to Mahiro style
- `Avoid` = likely drift
- `Why` = the intent behind the preference
