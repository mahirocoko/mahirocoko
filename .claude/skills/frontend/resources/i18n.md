# i18n (Lingui)

This guide standardizes i18n on Lingui.

## Rules

- Use `useLingui()` and `t` macro for plain translatable strings
- Use `Trans` for strings containing markup/components
- Avoid plain hard-coded UI strings in feature code
- Run locale extraction/compile command from the project runtime after adding new strings

## Example: Basic Translation

```tsx
import { useLingui } from '@lingui/react/macro'

export function HeaderTitle() {
  const { t } = useLingui()
  return <h1>{t`Dashboard`}</h1>
}
```

## Example: Markup Translation

```tsx
import { Trans } from '@lingui/react/macro'

export function TermsNotice() {
  return (
    <Trans>
      By continuing, you agree to the <strong>Terms and Conditions</strong>.
    </Trans>
  )
}
```

## Anti-Patterns

- `t('Hello')` without macro syntax
- String concatenation that breaks translation units
- Adding user-visible strings without extraction/compile step
