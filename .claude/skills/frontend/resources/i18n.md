# i18n (Lingui)

This guide standardizes i18n on Lingui.

## Core Rule: Source Language

**Always use the default language in `t` macro** — if `DEFAULT_LANG = 'th'`, write Thai. If `DEFAULT_LANG = 'en'`, write English.

```ts
// config.ts
export const DEFAULT_LANG = 'th'  // ← determines source language
```

```tsx
// ✅ ถูก - source เป็นภาษาไทย (default)
const { t } = useLingui()
return <h1>{t`เข้าสู่ระบบ`}</h1>

// ❌ ผิด - source เป็นภาษาอังกฤษ แต่ default เป็นไทย
return <h1>{t`Sign in`}</h1>
```

## Constants Don't Need Translation

App name, brand, or static identifiers should use constants directly — no `t` macro needed.

```tsx
// ✅ ถูก - ค่าคงที่ไม่ต้อง translate
import { APP_NAME } from '@/constants/config'
return <p>{APP_NAME}</p>

// ❌ ผิด - ไม่จำเป็น
return <p>{t`Haabiz HRM`}</p>
```

## Rules

- Use `useLingui()` and `t` macro for plain translatable strings
- Use `Trans` for strings containing markup/components
- **Source language = `DEFAULT_LANG`** — write in that language inside `t`
- Avoid plain hard-coded UI strings in feature code
- Run locale extraction/compile command from the project runtime after adding new strings

## Example: Basic Translation

```tsx
import { useLingui } from '@lingui/react/macro'

// Thai as source (DEFAULT_LANG = 'th')
export function HeaderTitle() {
  const { t } = useLingui()
  return <h1>{t`แดชบอร์ด`}</h1>
}

// English as source (DEFAULT_LANG = 'en')
export function HeaderTitle() {
  const { t } = useLingui()
  return <h1>{t`Dashboard`}</h1>
}
```

## Example: Markup Translation

```tsx
import { Trans } from '@lingui/react/macro'

export function TermsNotice() {
  return (
    <Trans>
      By continuing, you agree to the <strong>Terms and Conditions</strong>.
    </Trans>
  )
}
```

## Folder Structure

```text
app/
  constants/
    config.ts         # DEFAULT_LANG, SUPPORTED_LANGUAGES, APP_NAME
  libs/
    lingui/
      config.ts       # LinguiConfig (locales, catalogs)
      index.ts        # loadCatalog(), getLanguages()
  locales/
    th.po             # Thai translations (source if DEFAULT_LANG = 'th')
    en.po             # English translations
    *.js              # Compiled catalogs (gitignored)
  stores/
    setting/
      index.ts        # lang state with persist
      types.ts
```

## .gitignore Pattern

```gitignore
# Lingui compiled locales
app/locales/*.js
```

Only track `.po` files — compiled `.js` are generated artifacts.

## Provider Integration

```tsx
// app/providers/page-provider.tsx
import { I18nProvider } from '@lingui/react'
import { i18n } from '@lingui/core'
import { DEFAULT_LANG } from '@/constants/config'
import { loadCatalog } from '@/libs/lingui'
import { useSettingStore } from '@/stores/setting'

// Load default catalog on module init
loadCatalog(DEFAULT_LANG)

export function PageProvider({ children }: PropsWithChildren) {
  const lang = useSettingStore((state) => state.lang)

  // Reload catalog when language changes
  if (i18n.locale !== lang) {
    loadCatalog(lang)
  }

  return (
    <I18nProvider i18n={i18n}>{children}</I18nProvider>
  )
}
```

## Commands

```bash
# Extract + compile
pnpm lingui

# Or separate steps
lingui extract --clean --overwrite
lingui compile
```

## Anti-Patterns

- `t('Hello')` without macro syntax — use `t`Hello`` backticks
- String concatenation that breaks translation units
- Adding user-visible strings without extraction/compile step
- **Using English in `t` when `DEFAULT_LANG = 'th'`** — source must match default
- Translating constants like `APP_NAME` — they're intentionally static
