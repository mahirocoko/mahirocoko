# Spark needs a contained profile

When `gpt-5.3-codex-spark` is visible in provider caches and docs but still has access and parameter caveats, treat it as an experimental fast-path model, not as the new universal default. Keep high-confidence orchestration on `gpt-5.4`, keep durable coding paths on `gpt-5.3-codex`, and isolate Spark to lightweight categories or agents with explicit fallbacks.
