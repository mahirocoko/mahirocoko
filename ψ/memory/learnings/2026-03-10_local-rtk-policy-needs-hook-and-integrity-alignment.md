# Local RTK policy needs hook and integrity alignment

If a team wants a stricter local RTK rule than upstream RTK normalization, changing only the instruction text is not enough. The instruction layer and the hook layer must agree, otherwise the runtime behavior will continue to pull commands back toward upstream defaults. In this session the local doctrine was "preserve the repo-native command shape first, then prepend `rtk`", but `rtk rewrite` still collapsed `pnpm lint` into `rtk lint`. A local hook override fixed the functional behavior, yet that same patch triggered RTK's hook integrity check and blocked `rtk` execution. The durable lesson is that local RTK customization has three parts, not two: doctrine, runtime rewrite, and integrity/update model. If any one of those is ignored, the setup will feel correct on paper but unstable in real use.
