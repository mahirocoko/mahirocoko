# Skills-as-context beats model choice for frontend design

## Pattern

The quality gap between AI models for frontend design is smaller than the gap between "model with skill" vs "model without skill." A well-crafted markdown skill file (frontend-design doctrine) transforms even a model that defaults to purple-gradient slop into a producer of minimal, UX-aware, iterative designs.

## Evidence

- Opus 4.5 without skill = generic purple gradients (indistinguishable from other models)
- Opus 4.5 with skill = best-in-class frontend design with animation, UX consideration, and strong iteration
- Gemini 3 Pro has good defaults but can't iterate — it ignores refinement instructions
- Source: Theo - t3.gg (2026-02-05 video, watched 2026-03-15)

## Implication

Invest in skill quality, not model switching. Our existing `frontend-design` skill is the right approach. Iteration instructions in the skill are as important as aesthetic guidance — a model that can't iterate on feedback is limited regardless of first-shot quality.
