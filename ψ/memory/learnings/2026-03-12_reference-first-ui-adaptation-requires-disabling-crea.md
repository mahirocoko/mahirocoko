Reference-first UI adaptation works better when the workflow explicitly disables creative-first frontend guidance. In this session, the first `shadcn-template-lab` rewrite was technically coherent but visually wrong because a builtin frontend skill optimized for bold interpretation instead of fidelity. The correction only became possible after tracing that skill to upstream builtin behavior, disabling it in config, and re-running the UI pass with `shadcnspace` treated as a concrete source of truth instead of loose inspiration.

The practical pattern is: when the user wants “make this feel like that reference,” preserve structure if it is good, but rollback the visual thesis fast if the tone drifts. Fidelity comes more from section order, copy model, card anatomy, CTA geometry, and restraint than from simply matching colors. A landing page especially should never explain the redesign to the user; it should present a product proposition, category structure, proof, and clear browse paths.
