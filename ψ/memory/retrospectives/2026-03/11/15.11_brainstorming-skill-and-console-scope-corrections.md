## Honest Feedback

I made the same class of mistake more than once in this session: I let generic frontend heuristics outrun local intent. First it showed up in layout work, where I treated a reference image as permission to move farther away from the design system than the repo really wanted. Then it showed up again in token work, where a request to make the sidebar accent grayer became an unnecessary retune of base sidebar tokens. Then it appeared a third time in code shape, where I optimized for de-duplication by extracting class constants even though that was not the repo’s preferred form.

The important thing is that these were not random bugs. They were all the same failure mode: over-generalizing from a toolkit of “usually good” frontend moves. The user had to repeatedly narrow me back to the real contract. That means I need a stricter internal checkpoint when touching visual systems: ask whether the request is about structure, accent, or ownership before changing anything. It also means my verification needs to include code-shape review against live repo preference, not just diagnostics and visual plausibility.

## Lessons Learned

- Accent tuning is not license to retune the whole token family.
- Owner-local preference beats generic cleanup, especially in UI code shape.
- For this repo, readable UI class composition means `cn(...)` inline, not extracted one-off constants.
- Shell resets can be more valuable than iterative polish when the feature hierarchy is still unsettled.
- Skill bundles are strongest when they ship docs, runtime, and references together.

## Extracted Patterns

- **Skill bundle pattern**: `SKILL.md` + references + scripts + tool entrypoint is a durable packaging model.
- **Shell-first reset pattern**: when layout is confused, reduce each page to a shared placeholder and recover hierarchy later.
- **Preference-before-theory pattern**: follow explicit user correction first, then refine the doctrine from that correction.
- **Scope-discipline pattern**: a narrow request should stay narrow even if adjacent changes feel tempting.
