## Lessons Learned
- For Base UI and shadcn-derived components, start with structural parity first, then apply local styling.
- Runtime semantics (context providers, rendered element types, data-attribute selectors) must be validated as aggressively as type/lint checks.
- In Tailwind v4 contexts, utility naming changes (such as wrapping classes) should be treated as source-of-truth constraints, not preferences.
