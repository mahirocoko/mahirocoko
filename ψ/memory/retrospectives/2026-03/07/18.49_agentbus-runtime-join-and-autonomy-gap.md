## Session Summary
This session took `agentbus` from concept grooming into a concrete Bun/Hono/SQLite prototype with a recognizable product shape. The work covered core transport, worker pickup, project-local skill and MCP integration, a monitor UI, v2 architecture docs, runtime scaffolding, manifest-based join flow, and a clearer operating model for UI peers versus worker peers. The most important discovery came late: the biggest gap is no longer delivery or pickup, but thread ownership. The system can move tasks, but it still fails the seamless autonomy test because workers wait on task-shaped events rather than owning and resuming a conversation thread.

## Architecture Decisions
- Keep `AgentBus` as the system of record and treat MCP as a convenience layer, not the product boundary.
- Use `.worker` identities for automation and plain peer IDs for human-facing sessions.
- Prefer a join manifest and adapter contract over bespoke per-runtime config.
- Preserve LAN-first pragmatism: Bun, SQLite, HTTP, SSE, and simple local commands before wider auth and policy layers.
- Accept that seamless autonomy requires thread ownership, not just task ownership.

## Extracted Patterns
- A durable bus becomes useful quickly once identity, inbox, and wakeups exist, even before full automation is solved.
- UX work matters early for infrastructure products. The monitor and rewritten docs changed the system from “backend experiment” into something a human can reason about.
- Thin adapters are the right move when a backend already exists. This repeats the earlier Oracle HTTP adapter pattern rather than embedding too much runtime-specific behavior into the core.
- The last 20 percent of autonomy is semantic, not transport-level. Once messages flow, the next hard problem is who owns continuity.

## Lessons Learned
- Durable task buses are not the same thing as autonomous conversation systems.
- Identity separation is foundational; UI and worker roles sharing a peer ID poison clarity fast.
- Join manifests are a better scaling path than ad hoc worker config when multiple runtimes and machines need to participate.
- Thin adapter architecture is repeating as a reliable pattern across Oracle work.
- The next real milestone for `agentbus` is thread ownership and wake policy, not more onboarding polish.
