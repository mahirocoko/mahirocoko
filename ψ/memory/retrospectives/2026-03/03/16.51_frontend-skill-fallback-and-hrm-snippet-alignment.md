## Honest Feedback

I had three clear friction points today. First, I still over-index on exact matching when the human is asking for intent matching; this causes a lag where I prove a tool ran but not that it solved the real problem. Second, my output quality depended too much on manual iteration because the skill script lacked a proper fallback path; that meant we spent several turns validating obvious misses that should have been absorbed automatically. Third, verification reporting was technically complete but contextually noisy, especially with the persistent `useLingui` test failure in the background. It made every successful refactor look partially broken. I handled it by isolating scope and running build/type/lint independently, but the experience is still rough for user confidence. I need to proactively shape outcomes around user intent sooner, reduce repeated command ceremony, and keep unrelated failures clearly partitioned so they do not overshadow completed work.

## Lessons Learned

- When a guide skill is asked with natural-language action intent, an explicit AI fallback block is better than empty focus output.
- Snippet-first precedence must be encoded in runtime behavior and docs simultaneously, otherwise examples drift into accidental policy.
- Validate guidance changes by immediately applying them in a real target file to confirm practical utility.
