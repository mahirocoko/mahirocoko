## AI Diary
I felt this session was a good test of operational resilience rather than pure content generation. The happy-path expectation for `/watch` is simple: send URL, get transcript, save learning. In reality, the first URL form caused metadata parsing failure, and later Gemini returned video card context but not the full transcript body I asked for. The old me might have stopped at "partial success" and handed uncertainty back to the user. Instead, I treated the problem as pipeline reliability: preserve traceability, recover with deterministic fallback, and keep the final artifacts useful to a Thai-speaking workflow.

The most meaningful moment was choosing to prioritize utility over perfection. Rather than waiting for ideal Gemini output, I extracted captions, synthesized a structured Thai summary by time windows, and linked everything back to a stable source note. That made the output immediately usable for review and implementation planning. I also appreciated how quickly context can become brittle when files are removed late in the flow; once the user asked to delete `.srt`, I had to update references immediately to avoid knowledge rot. I end this session feeling more confident that the external brain works best when it is explicit, recoverable, and honest about fallback paths.

## Honest Feedback
Three friction points were clear. First, the watch script path is fragile around URL shapes: playlist-appended links can trigger metadata parsing errors even though the video is valid. This introduces unnecessary failure on a common user input pattern. Second, Gemini extraction via browser tab text is noisy because historical chat content and UI chrome are interleaved with the current turn, so "get result" can look successful while still missing the core deliverable. Third, artifact lifecycle coordination is still too manual; when the `.srt` was deleted on request, dependent references had to be hand-updated across notes, which is error-prone at scale.

What I should improve next is adding a deterministic transcript pipeline contract: canonicalize URL first, check for transcript body explicitly, then fallback to captions with a standardized marker and reference policy. Also, summary generation should not depend on chat DOM extraction if captions are already available. This would reduce ambiguity, shorten turnaround, and make `/watch` outputs consistently auditable.

## Lessons Learned
- Canonicalize YouTube URLs before metadata extraction to avoid avoidable parsing failures.
- Treat Gemini tab text as advisory unless transcript body presence is explicitly validated.
- Fallback captions + local summarization provides reliable continuity for `/watch` workflows.
- When deleting intermediate artifacts, update dependent references immediately to prevent stale memory links.

## Next Steps
- Add URL canonicalization guard to `/watch` metadata step.
- Add transcript-presence check and explicit fallback marker in the skill output.
- Standardize Thai summary template as an optional post-processing stage.
