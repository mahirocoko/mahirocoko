## Honest Feedback

Three friction points stood out. First, the tool workflow encouraged many micro-adjustments before a robust pattern was applied globally. We got there, but the path was longer than needed; when a user says “lock cell,” the fastest route should be an immediate reusable structure (cell containers), not repeated direct text tweaks. Second, context switching between visual edits and retrospective/deep-agent orchestration creates cognitive drag. The session still succeeded, but the handoff between “designer mode” and “historian mode” can feel abrupt, and that raises the chance of losing nuanced decisions if notes are not captured quickly. Third, temporary operation files accumulated as a side effect of iteration, and only later did we tighten ignore rules. This is harmless short term, but over many sessions it creates avoidable noise and weakens trust in `git status` as a clean signal. Next time, I should enforce temp-ignore hygiene at the first sign of generated helper files.

## Lessons Learned

1. Lock visual systems with invariant geometry early (`cell width`, `row gap`, fixed behavior) before aesthetic tuning.
2. Keep annotation outside color swatches; use dedicated label rows for readability and maintainability.
3. Convert foundation docs into rendered examples whenever possible; examples reduce interpretation drift.
4. Treat design tokens as APIs: preserve history, supersede intentionally, and document mapping decisions.
5. Capture learnings immediately after major visual sessions while rationale is still fresh.
